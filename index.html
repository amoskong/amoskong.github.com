买房到底买的是什么?

Date 2015-04-27 02:46

2013年底我终于决定买房了, 之前没有一个明确的驱动力量, 积蓄也不多, 也不想摊上房贷这事. 北京的直接跳过，在西安看了几次房后, 觉得坑太多, 老婆不要买了, 但考虑到最终还是要解决住房问题, 且婚前开销不大也是一个买房的好时机.

现在看来买房需要留意的事情很多, 缺乏经验很容易被坑, 要时刻有保护自己权益的意识. 签合同前认真阅读, 说不定某个条款里就隐藏着圈套.

户型不要被样板间蒙蔽, 样板间的装饰布局会隐藏很多设计的缺陷, 等交房了才傻眼. 难道开发商请的设计师都是傻子搞这么多缺陷? 开发商可能从其他角度考虑牺牲业主利益, 比如考虑外表对商务区的影响, 对住宅进行不合理户型调整.

不要相信任何口头承诺, 在合同或者补充协议里写清楚, 实在不行就要收据字条, 记着录音.

一边大的问题, 开发商会在合同里做好手脚, 规避法律风险, 所以一定要在合同里发觉有问题的地方. 比如"阳台不封闭", 后续需要自己掏钱封闭阳台; "集中供暖", 要明确是市政集中供暖还是小区集中供暖; "户型布局不以宣传页为准", 这个时候你就得仔细比对宣传材料和合同里描述的区别.

分析手机APP通信协议

Date 2014-12-04 01:33

首先在Fedora系统里安装squid, 然后启动squid服务, 默认监听端口为3128

# yum install -y squid
# service squid start

联接手机到无线网络, 设置在手机设置->WLAN里，修改所链接网络的设置，在高级选项里设置代理，代理IP地址为 Fedora 的IP地址, 端口为3128

然后在fedora上使用wireshark, tcpdump进行监听, 这里可以通过源IP和目标IP,协议端口等进行过滤, APP一般都是通过HTTP请求和后台进行数据通信.

# yum install -y wireshark-gnome
# sudo tcpdump -i wlp3s0 src 10.208.9.91 or dst 10.208.9.91 -A
# wireshark

修改mutt里GPG 签名公钥附件名

Date 2014-11-04 18:22

在mutt里配置好gpg后, 加密邮件会附带一个公钥文件, 这个文件名在Gmail里是 noname, 在QQ邮箱里是tcmime.1591.1634.2471.bin. 这些都会给人造成困惑, Thunderbird 里会把这个文件命名为 signature.asc

在archlinux 的github [1], 有一个patch为mutt添加了定义公钥文件名称和描述的功能. 于是从FedoraProject[2]下载mutt代码, 重新用rpmbuild编译rpm安装. 当然也可以使用Upstream[3]的代码, 记着configure时候打开需要的特性, 如 --enable-hcache 用于缓冲邮件头部改进加载速度.

安装完mutt, 只需要在~/.Muttrc里添加如下内容, 就可以修改公钥文件名了.

set pgp_mime_signature_filename="signature.asc" set pgp_mime_signature_description="Digital signature"
