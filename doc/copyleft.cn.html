What is Copyleft?

什么是 Copyleft？

Copyleft is a general method for making a program or other work free, and requiring all modified and extended versions of the program to be free as well.

Copyleft是一种使一个程序或其他工作自由的一般方法，并且要求所有对该程序的修改版本和衍生版本也是自由的。

The simplest way to make a program free software is to put it in the public domain, uncopyrighted. This allows people to share the program and their improvements, if they are so minded. But it also allows uncooperative people to convert the program into proprietary software. They can make changes, many or few, and distribute the result as a proprietary product. People who receive the program in that modified form do not have the freedom that the original author gave them; the middleman has stripped it away.

使一个程序成为自由软件的最简单的方式就是将它放到一个没有版权的公共领域。如果他们足够明智，它允许人们共享该程序和他们的进一步改进。但是它也允许不愿意合作的人们将程序转换转化为一个 专有软件。他们可以或多或少地做一些修改，同时将其作为专有软件发布，那些得到程序的人因为修改的形式而不能拥有最初作者给他们的自由，因为中间人已经将这种权利剥夺。

In the GNU project, our aim is to give all users the freedom to redistribute and change GNU software. If middlemen could strip off the freedom, we might have many users, but those users would not have freedom. So instead of putting GNU software in the public domain, we “copyleft” it. Copyleft says that anyone who redistributes the software, with or without changes, must pass along the freedom to further copy and change it. Copyleft guarantees that every user has freedom.

在GNU项目中，我们的目标是赋予所有使用者重新发布和修改GNU软件的自由。如果中间人剥夺了这种自由，我们可能会有一些使用者，但他们将不能享有这种自由。因此代替将GNU软件放到公共的领域，我们"Copyleft"它。Copyleft意味着 重新发布软件或没有任何修改的任何人必须传承这种自由以使人们进行更多的拷贝和修改，Copyleft保证每一个人都享有自由的权利。

Copyleft also provides an incentive for other programmers to add to free software. Important free programs such as the GNU C++ compiler exist only because of this.

Copyleft也为其他的程序员们去发布他们的自由软件提供一种激励，重要的自由软件如仅存的GNU C++编译器就是一个很好的例子。

Copyleft also helps programmers who want to contribute improvements to free software get permission to do that. These programmers often work for companies or universities that would do almost anything to get more money. A programmer may want to contribute her changes to the community, but her employer may want to turn the changes into a proprietary software product.

Copyleft也帮助那些给自由软件贡献成果的程序员们获得这样做的许可，这些程序员经常工作在一些能为他们提供更多报酬的公司或大学里。一名程序员可能想为社区贡献他的成果，但是他的雇主可能想将其成果应用到专有软件产品中。

When we explain to the employer that it is illegal to distribute the improved version except as free software, the employer usually decides to release it as free software rather than throw it away.

当我们向老板解释除了自由软件，发布其他改进版本的软件是非法的时候，雇主常常决定与其舍弃不用它，不如将其作为自由软件来发行它。

To copyleft a program, we first state that it is copyrighted; then we add distribution terms, which are a legal instrument that gives everyone the rights to use, modify, and redistribute the program's code or any program derived from it but only if the distribution terms are unchanged. Thus, the code and the freedoms become legally inseparable.

对于copyleft一个程序，我们开始规定它是有版权的；随后我们增加分发规则，这些规则允许每一个人都有权利去使用，修改和重修改程序的代码或者其派生程序的代码，但前提是这些分布规则没有改变。因此，这些代码和自由精神也就绝对合法了。

Proprietary software developers use copyright to take away the users' freedom; we use copyright to guarantee their freedom. That's why we reverse the name, changing “copyright” into “copyleft.”

专有软件的开发者利用版权来剥夺使用者的自由，而我们使用版权来保证他们的自由。这正是为什么我们颠倒这个单词，将"copyright"改为"copyleft"的原因。

Copyleft is a way of using of the copyright on the program. It doesn't mean abandoning the copyright; in fact, doing so would make copyleft impossible. The word “left” in “copyleft” is not a reference to the verb “to leave” — only to the direction which is the inverse of “right”.

"Copyleft"是一种使用对程序享有版权的一种方式。它不意味着放弃版权；事实上，那样做会使copyleft变得不可能。Copyleft里的单词left与参考动词”to leave “没有任何联系，只是说明与"right"相反的一种目的。

Copyleft is a general concept, and you can't use a general concept directly; you can only use a specific implementation of the concept. In the GNU Project, the specific distribution terms that we use for most software are contained in the GNU General Public License (available in HTML, text, and Texinfo format). The GNU General Public License is often called the GNU GPL for short. There is also a Frequently Asked Questions page about the GNU GPL. You can also read about why the FSF gets copyright assignments from contributors.

Copyleft是一个抽象的概念，所以你不能直接使用这个抽象的概念，你仅仅只能使用这个概念的一个具体的实现。在GNU项目中，为大多数软件所使用的具体的分发规则都包含在了 GNU的通用公共许可证(可用在HTML, text和 Texinfo格式)。GNU通用公共许可证经常简称为GNU GPL。这也就是那个 常见问题答案页面中的GNU GPL，你也可以阅读关于 为什么FSF从投稿者中得到转让证书。

An alternate form of copyleft, the GNU Lesser General Public License (LGPL) (available in HTML, text, and Texinfo format), applies to a few (but not all) GNU libraries. To learn more about properly using the LGPL, please read the article Why you shouldn't use the Lesser GPL for your next library.

一个copyleft的交替形式，较少的GNU宽通用公共许可证(可用于HTML,text和 Texinfo格式)，应用于一小部分(并非全部)的GNU函式库。为了学习更多关于适当使用LGPL，请阅读文章Why you shouldn't use the Lesser GPL for your next library.

The GNU Free Documentation License (FDL) (available in HTML, text and Texinfo) is a form of copyleft intended for use on a manual, textbook or other document to assure everyone the effective freedom to copy and redistribute it, with or without modifications, either commercially or noncommercially.

GNU自由文档许可证(FDL)(可用于HTML,text和Texinfo格式)是一种copyleft的形式，它有计划地使用手册，教科书或者其它文档以确保每个人有真正的自由去复制和重新发布它，而不管它是否修改或者是用于商业或非商业。

The appropriate license is included in many manuals and in each GNU source code distribution.

适当的许可证被包含在许多指南和每一个GNU的源代码的发行版中。

All these licenses are designed so that you can easily apply them to your own works, assuming you are the copyright holder. You don't have to modify the license to do this, just include a copy of the license in the work, and add notices in the source files that refer properly to the license.

所有的这些证书被设计成使你能容易将它们用于你自己的作品中。假定你是版权所有者，你不得不修改许可证来做这样的事情，仅仅包含一份许可证的拷贝在作品中，并且在源文件添加声明从而使它们能匹配适当的许可证。

Using the same distribution terms for many different programs makes it easy to copy code between various different programs. Since they all have the same distribution terms, there is no need to think about whether the terms are compatible. The Lesser GPL includes a provision that lets you alter the distribution terms to the ordinary GPL, so that you can copy code into another program covered by the GPL.

针对许多不同的程序使用相同的分发规则可以使得它能在不同的程序之间容易地复制代码。既然都是用相同分发规则, 这就没有必要去考虑这些分发规则是否合适自从它们有了相同的分发条款对普通的GPL，宽通用公共许可证包含一个让你依据普通GPL更改的规则，从而使你能依据GPL将你的代码拷贝到另一个被覆盖的程序中。

If you would like to copyleft your program with the GNU GPL or the GNU LGPL, please see the license instructions page for advice. Please note that you must use the entire text of our licenses, if you use one. Each is an integral whole, and partial copies are not permitted.

如果你想用GNU GPL或GNU LGPL来copyleft你的程序，请查看用于建议的许可证的规程页面。请注意你必须使用我们的许可证的整个文本，不管你是否只使用一个。每一个都是一个固有的整体，因此部分的拷贝是不被允许的。

If you would like to copyleft your manual with the GNU FDL, please see the instructions at the end of the FDL text, and the GFDL instructions page. Again, partial copies are not permitted.

如果你用GNU FDL来copyleft你的手册指南，请查看FDL文本中 结尾的规程和 GFDL规程。同样的，部分的拷贝是不被允许的。
